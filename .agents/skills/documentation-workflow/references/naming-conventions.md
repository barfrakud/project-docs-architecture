# Naming Conventions

## General
- Use `kebab-case`
- Prefer short, descriptive names
- Avoid spaces and special characters

## Plans, prompts, reports
Use:
`YYYY-MM-DD-topic.md`

Examples:
- `2026-03-13-cicd-migration-plan.md`
- `2026-03-13-codex-docs-prompt.md`
- `2026-03-13-cicd-implementation-report.md`

## Manuals
Use task-oriented names:
- `install-docker-on-ubuntu.md`
- `configure-nginx-reverse-proxy.md`
- `troubleshoot-docker-compose-network.md`

## Decisions
Use ADR-like numbering when helpful:
- `001-use-docker-compose.md`
- `002-use-jenkins.md`
