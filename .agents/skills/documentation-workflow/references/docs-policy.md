# Documentation Policy

## Private-first policy

All exploratory, educational, or execution-level documentation starts in `docs/private/`.

This includes:
- implementation plans,
- prompts,
- reports,
- research notes,
- concept notes,
- manuals,
- architecture decisions.

## Public promotion policy

Public documentation must be a distilled version of private materials.

Before promoting content into `docs/public/`:
- remove scratch reasoning,
- remove learning-only detail,
- remove internal uncertainty unless it matters,
- keep the tone concise and professional.

## Minimum expected outputs

For meaningful implementation work:
- plan,
- prompt if AI was used significantly,
- report.

## Durable knowledge destinations

- procedures -> `docs/private/manuals/`
- concepts -> `docs/private/concepts/`
- decisions -> `docs/private/decisions/`
