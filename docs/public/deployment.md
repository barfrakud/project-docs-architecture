# Deployment

This repository is meant to be published as a standard Git repository.

## Recommended publication models

### Model A: separate private and public repositories
Keep complete working documentation in a private repo.
Publish a cleaned public copy that includes:
- `README.md`
- `docs/public/`
- optional templates and public examples

### Model B: one repository with selective publication
Keep private docs locally or in a protected branch, and publish only the public-facing subset.

## Recommendation

For portfolio and consulting use, **separate private and public repositories** is usually the cleanest option.
